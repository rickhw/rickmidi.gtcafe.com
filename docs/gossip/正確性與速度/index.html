正確性與速度

幾個月前工作上遇到一個問題，就是正確性與速度 (效率) 的課題。

我負責開發一個 USB Device 的通訊協定，上面期望的結果是既正確，又快速，讓我很為難。這種 Device 工作方式是 Polling (輪詢) 機制，Device 本身的 micro-chip 處理速度就不快了，加上 Polling 要額外付出的尋訪時間，所以要正確又要快速，兩者我只能取其一；正確就快不起來，要快 (做個樣子) 就無法正確。

這跟音樂有啥關係？是有的，音樂的演奏技術是建立在正確的表達之上，不管是速度，還是多麼花俏的技巧，多麼炫麗的音色，這些東西都必須建立在正確的彈奏之上。彈的又快又好，要先建立在彈的又慢又穩之上。歌曲的速度 200，那麼就要先把 100 練好；想要在速度 200 上彈一些花俏的技巧，那就先把 100 練好，然後慢慢往上增加。

所以以吉他來說，當發現練習的曲子會卡卡的，不順的時候，很多時候，除了基本的技巧不夠紮實 (吉他的悶放音、搥勾弦、推弦、顫音 …)。另外可能就是練習的速度太快了，超過自己手能夠反應的速度，以致於沒有辦法清楚精確的把樂句彈好。這時候就把速度放慢，重新仔細檢視每一個音符，重新確認每一個表情都是否能夠彈在對的拍點上，都確認了，然後一直練習到可以自行加減表情，加減音符的時候，就可以往下一個速度邁進了。

至於說如何檢視自己練習的成果，錄音是一個很好的方式。因為寫這篇文字的時候，個人簡易的錄音設備已經滿街都是。各大廠從小型到大型的綜合效果器、簡易的 USB 錄音介面，都是小而美的東西，所以不要裝死，認真的面對自己彈奏的東西，認真的為自己彈出的每個音符負責。至於吉他練習的方式，就看看本書的 “吉他基本功”。

台語說 “吃快撞破碗”，大概就是這個道理。所以其實練習的過程也是在練耐心、定性。

當然那個任務，最後成功的說服了上面。因為如果給客戶的東西是，很有效率，但是結果都不正確，跟結果都正確，但是速度慢了一點，我想客戶應該能接受的是後者吧。
